# 多平台内容解析思路与实现指南

## 核心解析思路

所有平台的内容解析遵循相同的**网页数据挖掘**模式：

```
分享链接 → 重定向处理 → 获取页面HTML → 提取JavaScript数据 → 解析结构化信息
```

### 通用流程步骤

1. **链接处理**：处理分享链接的重定向，获取真实页面URL
2. **页面获取**：使用合适的User-Agent获取完整HTML页面
3. **数据提取**：从页面中的JavaScript变量提取JSON数据
4. **信息解析**：将原始数据解析为统一的结构化格式（`{ project, author, statistics, platform }`）

## 已实现平台对比

### 抖音 (Douyin)

抖音分享链接需经过两次跳转：先跟随分享链接拿到真实地址，再利用 `iesdouyin.com` 构造标准页面 URL 来抓取。

```javascript
// 数据源位置
window._ROUTER_DATA

// 数据路径（二选一）
loaderData['video_(id)/page'].videoInfoRes      // 视频
loaderData['note_(id)/page'].videoInfoRes        // 图集（抖音笔记）

// 最终数据
videoInfoRes.item_list[0]

// 内容类型判断
aweme_type: 4  // 视频
aweme_type: 2  // 图集/图文

// 关键字段结构
{
  aweme_id: "作品ID",
  desc: "作品描述",
  aweme_type: 2 | 4,
  author: {
    nickname: "用户昵称",
    short_id: "抖音号",
    avatar_thumb: { url_list: ["头像地址"] }
  },
  video: {
    play_addr: { url_list: ["视频地址"] },  // 含 playwm 需替换为 play
    cover: { url_list: ["封面地址"] }        // 含 playwm 需替换为 play
  },
  images: [{ url_list: ["图片地址"] }],       // 仅图集类型有此字段
  statistics: {
    digg_count: 点赞数,
    comment_count: 评论数,
    share_count: 分享数,
    collect_count: 收藏数
  }
}
```

**关键实现细节：**
- 视频 URL 中的 `playwm` 替换为 `play` 可去水印
- 图集无 `video` 字段，使用 `images` 数组
- 作者头像通过正则替换 `/{width}x{height}/` 为 `/720x720/` 统一尺寸
- 需要同时尝试 `video_(id)/page` 和 `note_(id)/page` 两种 loaderData 路径

### 小红书 (Xiaohongshu)

小红书分享链接使用 `xhslink.com` 短链重定向，笔记 ID 可从 `/explore/`、`/discovery/item/` 或 `/note/` 路径中提取。

```javascript
// 数据源位置
window.__INITIAL_STATE__

// 数据路径
note.noteDetailMap[noteId].note

// 内容类型判断（无统一类型字段，需从 video 对象是否存在判断）
type: "video"        // 视频笔记（有 video 子对象）
type: "normal"       // 图文笔记（无 video 子对象）

// 关键字段结构
{
  noteId: "笔记ID",
  title: "笔记标题",
  desc: "笔记描述",
  type: "video" | "normal",
  user: {
    userId: "用户ID",
    nickname: "用户昵称",
    avatar: "用户头像URL"
  },
  imageList: [{ urlDefault, urlPre, url }],
  video: {
    media: {
      stream: {
        h264: [{ masterUrl: "视频地址" }]
      }
    }
  },
  tagList: [{ name: "标签名", type }],
  interactInfo: {
    likedCount: 点赞数,
    collectedCount: 收藏数,
    commentCount: 评论数,
    shareCount: 分享数
  }
}
```

**关键实现细节：**
- JSON 数据中可能包含 `undefined` 值，需替换为 `null` 再解析
- 视频内容判断：`noteData.type === 'video'` 或 `noteData.video?.media` 存在
- 视频封面取 `imageList[0].urlDefault`
- 图片地址优先级：`urlDefault > urlPre > url`
- 标题为空时降级使用 `desc` 作为标题
- 统计字段是字符串类型，需要 `parseInt` 转换

## 架构设计

### 分层架构

```
routes/video.js              ← HTTP 路由层
  → platformProcessor.js     ← 调度层（策略模式）
    → platforms/*Parser.js   ← 各平台解析器
```

### 调度层 (PlatformProcessor)

- `detectPlatform(url)` — 根据域名识别平台
- `parseShareUrl(shareText)` — 从分享文本中提取链接，识别平台，调用对应解析器
- 新平台需：创建解析器 → 注册到 `this.parsers` → 添加 `detectPlatform` 规则

### 代理层

解析结果中的 `url_list` 和 `cover` 字段在 API 层会被递归包装为同源代理地址：

```javascript
// 在 routes/video.js 中
const proxyBase = `${ctx.origin}/api/proxyFile?url=`
wrapUrlsWithProxy(videoInfo, proxyBase)
```

这样前端无需处理跨域问题，所有资源通过同一个域名访问。

## 新增平台适配指南

### Step 1: 创建解析器

在 `src/utils/platforms/` 下创建 `{platform}Parser.js`：

```javascript
const axios = require('axios')
const { getRandomUA } = require('../userAgents')

class NewPlatformParser {
  async parseUrl(shareUrl) {
    // 1. 跟随重定向获取真实链接
    const shareResp = await axios.get(shareUrl, { headers: getRandomUA() })
    const realUrl = shareResp.request.res.responseUrl

    // 2. 提取内容ID
    const idMatch = realUrl.match(/\/path\/([\w]+)/)
    if (!idMatch) throw new Error('无法提取内容ID')
    const contentId = idMatch[1]

    // 3. 获取页面HTML
    const resp = await axios.get(realUrl, {
      headers: {
        ...getRandomUA(),
        'Accept': 'text/html,application/xhtml+xml,application/xml;q=0.9,image/webp,*/*;q=0.8',
        'Accept-Language': 'zh-CN,zh;q=0.8,zh-TW;q=0.7,zh-HK;q=0.5,en-US;q=0.3,en;q=0.2'
      }
    })

    // 4. 提取 JavaScript 数据源
    const html = resp.data
    const scriptMatch = html.match(/window\.__DATA_SOURCE__\s*=\s*({.*?})\s*<\/script>/s)
    if (!scriptMatch) throw new Error('无法提取页面数据')

    // 5. 解析 JSON（注意清理 undefined/尾逗号）
    let jsonStr = scriptMatch[1]
    jsonStr = jsonStr.replace(/undefined/g, 'null')
    const jsonData = JSON.parse(jsonStr)

    // 6. 定位具体内容数据（根据平台结构调整路径）
    const contentData = jsonData.path?.to?.data?.[contentId]
    if (!contentData) throw new Error('未找到内容数据')

    // 7. 转换为统一格式
    return this.parseData(contentData, contentId)
  }

  parseData(rawData, contentId) {
    // 类型判断
    const isVideo = rawData.type === 'video' || !!rawData.video

    // 提取媒体资源
    let videoUrl = null
    let coverImg = []
    let allImg = []
    if (isVideo) {
      videoUrl = rawData.video?.playUrl || null
      if (rawData.cover) coverImg = [rawData.cover]
    } else if (rawData.images?.length) {
      allImg = rawData.images.map(i => i.url).filter(Boolean)
    }

    return {
      project: {
        project_id: contentId,
        title: (rawData.title || `platform_${contentId}`).replace(/[\\/:*?"<>|]/g, '_'),
        desc: rawData.desc || '',
        type: isVideo ? 'video' : 'image',
        cover: coverImg[0] || '',
        url_list: isVideo ? (videoUrl ? [videoUrl] : []) : allImg,
      },
      author: {
        author_id: rawData.author?.id || '',
        nickname: rawData.author?.nickname || '未知用户',
        avatar: rawData.author?.avatar || '',
      },
      statistics: {
        digg_count: parseInt(rawData.stats?.likes) || 0,
        comment_count: parseInt(rawData.stats?.comments) || 0,
        share_count: parseInt(rawData.stats?.shares) || 0,
        collect_count: parseInt(rawData.stats?.collects) || 0,
      },
      platform: 'new-platform',
    }
  }
}

module.exports = NewPlatformParser
```

## 常见问题

### 1. JSON.parse 失败

- **原因**：页面数据中包含 JS 特有值（`undefined`、尾逗号）
- **解决**：替换 `undefined` → `null`，去除尾逗号

### 2. 反爬虫拦截

- **策略**：使用移动端 User-Agent（iPhone Edge/微信浏览器），添加 `Accept-Language: zh-CN`

### 3. 数据路径变化

- **策略**：支持多条数据路径降级，如抖音同时尝试 `video_(id)/page` 和 `note_(id)/page`

### 4. 内容类型误判

- **策略**：多条件判断，如图文同时检查 `type` 字段和 `video` 子对象是否存在
